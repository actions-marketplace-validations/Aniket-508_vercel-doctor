## vercel-edge-heavy-import

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-edge-heavy-import`</Callout>

Detecta arquivos com runtime edge importando dependﾃｪncias pesadas ou centradas em Node como `node:fs`, `node:crypto`, `sharp` ou `@aws-sdk/*`.

**Por que importa:** Funﾃｧﾃｵes edge tﾃｪm limites rigorosos de tamanho e execuﾃｧﾃ｣o. Dependﾃｪncias pesadas do Node.js aumentam o tempo de cold start e podem falhar em tempo de execuﾃｧﾃ｣o.

**Correﾃｧﾃ｣o:** Mova a lﾃｳgica pesada para funﾃｧﾃｵes com runtime Node.js ou jobs em segundo plano, e mantenha os handlers edge leves.

## vercel-sequential-database-await

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-sequential-database-await`</Callout>

Detecta rotas de API com 3 ou mais chamadas sequenciais ao Prisma ou banco de dados sem `Promise.all`.

**Por que importa:** Cada chamada sequencial ao banco de dados adiciona latﾃｪncia ﾃ execuﾃｧﾃ｣o da funﾃｧﾃ｣o. Paralelizar consultas independentes reduz a duraﾃｧﾃ｣o total e o custo.

```ts title="Bad"

const posts = await prisma.post.findMany();
const tags = await prisma.tag.findMany();
```

```ts title="Good"
const [users, posts, tags] = await Promise.all([
  prisma.user.findMany(),
  prisma.post.findMany(),
  prisma.tag.findMany(),
]);
```

## vercel-large-static-asset

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-large-static-asset`</Callout>

Detecta assets estﾃ｡ticos (imagens, fontes, vﾃｭdeos, PDFs) de 4 KB ou maiores servidos a partir do repositﾃｳrio do seu app.

**Por que importa:** Arquivos estﾃ｡ticos grandes servidos pelo seu deploy na Vercel consomem largura de banda em cada requisiﾃｧﾃ｣o. Movﾃｪ-los para um CDN dedicado ou armazenamento de objetos (Cloudflare R2, S3) reduz custos de largura de banda.

Reporta atﾃｩ 20 arquivos, ordenados por tamanho (maior primeiro).

## vercel-consider-bun-runtime

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-consider-bun-runtime`</Callout>

Detecta projetos nﾃ｣o configurados para o runtime Bun (sem `packageManager: "bun@..."` no `package.json` e sem arquivo `bun.lock`).

**Por que importa:** O runtime Bun pode reduzir a sobrecarga de instalaﾃｧﾃ｣o e build na Vercel comparado ao Node.js.

**Correﾃｧﾃ｣o:** Consulte a [documentaﾃｧﾃ｣o do runtime Bun](https://vercel.com/docs/functions/runtimes/bun) e mude se seu projeto for compatﾃｭvel.

## vercel-avoid-platform-cron

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-avoid-platform-cron`</Callout>

Detecta `crons` configurados no `vercel.json`.

**Por que importa:** Cron jobs da Vercel executam como funﾃｧﾃｵes serverless, cobrados por execuﾃｧﾃ｣o. Cargas de trabalho agendadas com padrﾃｵes previsﾃｭveis podem frequentemente executar de forma mais barata usando GitHub Actions ou Cloudflare Workers Cron Triggers.

## vercel-consider-fluid-compute

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-consider-fluid-compute`</Callout>

Detecta projetos com 3 ou mais rotas de API/servidor.

**Por que importa:** O Fluid Compute melhora a concorrﾃｪncia e reduz a sobrecarga de execuﾃｧﾃ｣o para cargas de trabalho com latﾃｪncia variﾃ｡vel ou trﾃ｡fego em rajadas. Vale a pena avaliar para projetos com mﾃｺltiplas rotas de servidor.

## vercel-suggest-turbopack-build-cache

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-suggest-turbopack-build-cache`</Callout>

This check is version-aware and only applies to Next.js 16+ projects.

Detecta arquivos `next.config` com configuraﾃｧﾃｵes `experimental` mas sem `turbopackFileSystemCacheForBuild`.

**Por que importa:** O Next.js 16+ suporta cache de build do Turbopack, que pode reduzir significativamente os tempos de build.

```js title="Good"
// next.config.js
module.exports = {
  experimental: {
    turbopackFileSystemCacheForBuild: true,
  },
};
```

## vercel-suggest-deploy-archive

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-suggest-deploy-archive`</Callout>

Detecta projetos com 5.000 ou mais arquivos.

**Por que importa:** Projetos grandes podem atingir limites de taxa da API durante o deploy. Usar o modo de arquivo faz upload de um ﾃｺnico tarball em vez de arquivos individuais, reduzindo o tempo de deploy em aproximadamente 50%.

```bash

```
